# 历史 Bug 案例库

这里存放已经修复、但值得长期保留的 Bug 回归案例。

## 维护原则

* 新发现的重要 Bug，修复后优先沉淀为 JSON 案例。
* 案例只描述最小复现，不重写业务逻辑。
* 继续沿用统一格式：
  * `name`
  * `description`
  * `input`
  * `expected`
* 需要复现的目标优先复用真实业务代码，不做测试专用分支。

## 推荐放置方式

* `database/`：数据库、事务、锁竞争、隔离性问题
* `bot/`：Bot 消息、分流、风控相关问题
* `score/`：计分导入、字段边界、特殊成绩问题
* `tournament/`：赛事创建、报名、结算、排名流程问题

`tests/run_all.py` 会自动执行这里的案例。
