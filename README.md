# 2048 赛事管理系统

这是一个面向 2048 赛事流程管理的项目，覆盖建赛、报名、提交、审核、结算、导出、锁局、评分与查询等核心能力。

## 快速开始

1. 安装项目依赖，按本地环境准备运行配置。
2. 如果需要外部集成，先复制 `.env.bot.example` 再填写本地配置，不要把真实密钥提交进仓库。
3. 使用 `data/testing.db` 作为默认测试库，不要把测试入口指向正式数据。
4. 跑完整测试：`python tests/run_all.py`
5. 快速跑测试：`python tests/run_all.py --fast`

## 仓库结构

- `services/`：核心业务逻辑
- `tests/`：回归、快照、回放与模拟测试
- `docs/`：公开文档与成绩模板
- `计分器/`：独立的计分与看板原型
- `config/`：配置相关代码
- `scripts/`：辅助脚本

## 维护约定

- 默认只使用 `data/testing.db` 做测试。
- 不要并行运行会重建 `testing.db` 的入口。
- 新问题优先修复，再补对应的回归案例。
- 能复用现有业务逻辑和快照的地方，优先复用，不要轻易重写计分、排名或结算逻辑。
