评分与长期统榜说明

一、评分的分层

当前项目里，评分/榜单不应只看单场结果，而是分成三层：

1. 原始成绩层
- 记录一局的原始结果。
- 对应 raw_score / final_score / competition_score / primary_time_ms 等字段。

2. 赛事层
- 记录某个 event 中，选手最终参与排名与结算的成绩。
- 同一名选手在同一赛事里可能会有多个候选记录。

3. 长期统榜层
- 记录跨赛事、跨时间窗口的长期表现。
- 不是单场结算的直接复制。

二、长期统榜的基本原则

1. 统榜要稳定
- 不要因为一次临时导入或一次锁局波动就大幅改变长期排行。

2. 统榜要可解释
- 看到某个分数时，要能追溯到：
  - 这是谁的
  - 来自哪场比赛
  - 当时用的是什么口径

3. 统榜要尽量少手工修正
- 能通过规则和来源数据推导的，尽量不要靠人工补表。

三、推荐口径

对于不同比赛类型，建议口径如下：

- 经典4x4 / Stone / 原始分赛：
  - 以 raw_score 或 final_score 为主
  - 一般按高分优先

- 限时赛 / 竞速赛：
  - 以 primary_time_ms 或目标达成时间为主
  - 一般按更低更好

- 特殊综合赛：
  - 先看比赛规则文档里的 ranking_metric
  - 再决定是分数型还是时间型

四、与代码的对应关系

主要参考：

- services/rating_service.py
- services/interim_ranking_service.py
- services/settlement_service.py
- config/rating_families.json
- config/rating_buckets.json

五、维护建议

1. 新增一个比赛模式前，先决定长期统榜是否纳入。
2. 如果要纳入，最好先定义：
   - 评分字段
   - 排序方向
   - 缺局处理
   - 平局处理
3. 不要让统榜口径和单场赛事结算口径悄悄分叉。
4. 发现口径分叉时，优先写清楚并统一规则，而不是重复人工补数据。
