# docs 目录约定

这份文件补充项目级约定，专门约束事故记录、交接文档和历史归档的维护方式。

## 当前优先级

- `docs/bug_history.md`：记录仍在关注的问题。
- `docs/bug_archive.md`：记录已确认稳定、无需继续盯的问题。
- `docs/incident_timeline.md`：记录事故时间线索引。
- `docs/agent_handoff.md`：记录给下一位 Agent 的接手说明。

## 写法原则

- 只记录对维护有价值的内容，不堆无关流水账。
- 新事故先写 `bug_history.md`，不要直接写进 archive。
- 已稳定的问题再迁移到 `bug_archive.md`。
- 事故脉络尽量保持时间一致、标题稳定、方便检索。

## 维护规则

- 重点关注账号稳定性、群聊风控、数据库锁竞争和消息链路抖动。
- 发现新问题时，先写现象、触发条件、根因、修复和当前状态。
- 如果问题会影响测试、回放或 bot 可用性，优先更新事故记录，再补对应回归。
- 交接文档优先写“现在还要盯什么”，不要写成冗长说明书。

## 归档规则

- 连续观察未复现且无需继续关注的问题，迁移到 `bug_archive.md`。
- 已归档内容尽量简短，只保留足够的历史线索。
- `incident_timeline.md` 保持索引短，月度明细放到对应子文件。
