比赛模式扩展说明

一、当前目标

把不同比赛模式的创建逻辑集中到模式目录里，避免以后每加一种模式都复制一套主办方菜单代码。

当前模式目录文件：

services/competition_mode_service.py

二、已接入主办方新建比赛的模式

1. 经典 4x4 原始分

competition_type = classic_raw_score
platform = 2048verse
variant = 4x4
rating bucket = classic_4x4_raw_score
排序：分数越高越好
成绩来源：Verse 锁局为主，也可导入补充成绩文件

2. Stone 4x4

competition_type = stone_x
platform = taihe
variant = 4x4
rating bucket:
stone_1k_4x4
stone_2k_4x4
排序：competition_score 越高越好
成绩来源：成绩文件导入

3. No-X 4x4

competition_type = no_x
platform = taihe
variant = 4x4
rating bucket:
no_1k_4x4
no_2k_4x4
no_8k_4x4
no_16k_4x4
排序：competition_score 越高越好
成绩来源：成绩文件导入

4. Speedrun 4x4

competition_type = speedrun
platform = taihe
variant = 4x4
rating bucket:
speedrun_1k_4x4
speedrun_2k_4x4
排序：primary_time_ms 越低越好
成绩来源：成绩文件导入

5. 斐波那契原始分

competition_type = fibonacci_raw_score
platform = taihe
variant:
fibonacci_3x3
fibonacci_4x4
fibonacci_5x5
rating bucket:
fibonacci_3x3_raw_score
fibonacci_4x4_raw_score
fibonacci_5x5_raw_score
规则说明：1 和 1 可合成 2，2 和 3 可合成 5，3 和 5 可合成 8，依斐波那契序列推进
排序：raw_score/final_score 越高越好
成绩来源：成绩文件导入

competition_score

Speedrun：
primary_time_ms
target_tile_value

raw_score 或 final_score

五、后续新增模式的做法

1. 如果只是新增目标值，先在 config/rating_buckets.json 增加 bucket。
2. 在 services/event_admin_service.py 的 infer_rating_bucket_code 增加映射。
3. 在 services/competition_mode_service.py 增加或扩展模式定义。
4. 如果排序方向不同，确认 event_admin_service.default_rule_values 里的 ranking_order。
5. 如果成绩字段不同，扩展 raw_score_file_parser 和 settlement_service。
