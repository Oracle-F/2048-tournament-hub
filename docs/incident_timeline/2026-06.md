## 2026-06-03 | QQ 私聊 bot 启动依赖缺失导致无法再次启动

* 发现时间：2026-06-03 上午
* 风险等级：低
* 触发入口：`run_bot.bat` / `scripts\run_private_qq_bot.py`
* 现象：
  * bot 在启动阶段就因缺少 `nonebot` 依赖而报错。
  * 私聊 bot 无法重新拉起，等同于入口不可用。
* 止损动作：
  * 先确认是启动链路报错，不继续在业务层反复重试。
  * 统一检查 Python 解释器与依赖安装状态。
* 排查过程：
  * 直接导入 bot 入口时发现 `nonebot` 模块缺失。
  * 确认启动脚本和自动启动脚本需要使用同一套解释器。
  * 发现问题属于运行环境不一致，而不是业务逻辑本身。
* 修复动作：
  * 将 `requirements-bot.txt` 安装到实际启动的 Python 3.12 环境。
  * 修正 `run_bot.bat` 与自动启动脚本，统一指向可用解释器。
  * 重新验证 bot 入口导入成功。
* 结果：
  * bot 启动链路恢复。
  * 入口可以再次拉起私聊 bot。
* 后续观察：
  * 后续若重装 Python 或迁移机器，优先核对 bot 依赖是否仍在同一解释器中。
  * 尽量保持启动脚本与依赖安装路径一致，减少环境漂移。

## 2026-06-02 | Codex Windows app 重置前启动异常与任务栏旧入口失效

* 发现时间：2026-06-02 晚间
* 风险等级：低
* 触发入口：Windows 任务栏固定图标 / 应用启动入口
* 现象：
  * 启动时弹出 `Error launching app`。
  * 提示无法找到 `OpenAI.Codex_...` 对应的 Electron app，并把 `type=click&tag=...` 一类参数当成路径。
  * 应用设置里的“修复”持续提示稍后重试。
  * 问题跨重启仍然存在。
* 止损动作：
  * 先退出 Codex。
  * 备份 `C:\Users\oracl\.codex`。
  * 暂不直接清理本地历史。
* 排查过程：
  * 确认这不是一次性卡死，而是持续性的启动异常。
  * 观察到任务栏固定入口也沿用了旧的启动信息。
  * 推测是更新后旧入口或应用注册状态没有刷新。
* 修复动作：
  * 使用 Windows 应用“重置”清除异常状态。
  * 重置后从开始菜单重新打开 Codex。
  * 重新固定新的可用任务栏入口。
* 结果：
  * 应用恢复正常。
  * 对话历史和本地状态未丢失。
* 后续观察：
  * 以后优先使用重置后新生成的入口。
  * 若再次出现同类报错，先检查旧 pin 和应用注册，再判断是否需要重置。
